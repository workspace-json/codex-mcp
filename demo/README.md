# Demo narration production

This directory is the repository-native source for the under-three-minute judge
demo narration: script, scene targets, voice settings, pronunciation guidance,
audio segments, timing, and captions. The committed sources are a strong draft,
not the final recording. HAC-98 and HAC-136 must supply preserved fixture and
reviewer evidence before the two explicitly marked placeholders are replaced.

## Source and boundaries

- `narration/script.md` is the human-reviewable script and on-screen replacement
  guide.
- `narration/scenes.json` is the canonical machine-readable source. Its `text` is
  exact spoken and captioned wording; `displayText` preserves product spelling for
  visuals. Scene status and `evidenceDependency` keep unfinished proof visible.
- `narration/pronunciations.json` records safe spoken substitutions without a
  server-side pronunciation-dictionary credential or ID.
- `narration/voice.example.json` is a checked-in provisional configuration. The
  authorized voice ID and API key are environment variables, never source files.

The voice is provisional. Use only a voice you own or are authorized to use; do
not imitate an identifiable person without rights. The README, script, and
captions preserve the boundary that the reviewer is read-only/advisory and that
missing evidence is unknown, not safe.

## Generate a draft

Requirements: Node 20+, `ffmpeg`/`ffprobe`, and an ElevenLabs account with an
authorized voice. The generator uses ElevenLabs' current HTTP text-to-speech
stream endpoint with `ELEVENLABS_API_KEY` and `ELEVENLABS_VOICE_ID`; it does not
add an SDK dependency or read a `.env` file.

```bash
export ELEVENLABS_API_KEY="..."
export ELEVENLABS_VOICE_ID="..."
node demo/narration/generate.mjs
node demo/narration/concatenate.mjs
node demo/narration/generate-captions.mjs
```

Run `node demo/narration/generate.mjs --force` to regenerate every scene. Without
`--force`, existing deterministic scene filenames are reused, so a revised line
can be regenerated by removing only its ignored output file and running the
generator again. Then rerun concatenation and captions.

Generated audio and manifests live in `narration/output/` and are ignored. The
manifests preserve per-segment and delivery hashes, actual durations, word count,
rate, and per-segment peak checks; they never contain the API key. `narration.wav` is the lossless
intermediate and `narration.mp3` is the delivery copy. This pipeline deliberately
does not promise byte-identical audio: current ElevenLabs synthesis can vary
between calls.

## Review before recording lock

1. Approve or reject the provisional voice, model, and settings.
2. Approve the draft narration script and its controlled-fixture disclosure.
3. After HAC-98/HAC-136 evidence exists, replace the two placeholders during
   HAC-160, regenerate affected segments, and inspect timing/captions again.

The concatenation command rejects a track over 2:58 and warns outside 2:35–2:52.
Keep the final track within 2:35–2:52 and never exceed 2:58. Leave the configured
silence after the deterministic denial, passing verification, reviewer verdict,
and final lockup so terminal evidence remains readable.
